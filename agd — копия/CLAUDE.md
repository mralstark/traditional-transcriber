# CLAUDE.md — транскрипция и редактура 15 бесед о традиционных ценностях (часть 2)

Это операционная инструкция для агента (Claude Code). Следуй ей по умолчанию; при конфликте с разовой просьбой пользователя — уточняй.

## Что это за проект

15 аудиозаписей на русском (по ~20–30 мин, ровно 3 спикера) → издательский связный текст. Делаем **только вторую часть** курса: традиционные ценности, которых **нет** в Указе № 809 (ценности из Указа уже изданы отдельной брошюрой и сюда не входят). Последовательность бесель содержательная и **не перемешивается**.

Спикеры (важны для биасинга ASR и глоссария):
- **Константин Малофеев** — бизнесмен, монархист, «Царьград». Лексика: монархия, самодержавие, империя, Катехон, Третий Рим, Византия, соборность, суверенитет, глобализм, культурный код.
- **Александр Дугин** — философ-традиционалист. Самая трудная речь для ASR: премодерн/модерн/постмодерн, традиционализм, эсхатология, Логос/мифос, Dasein (Дазайн), Хайдеггер, Генон, Эвола, Мирча Элиаде, Ноомахия, апофатика, сакральное/профанное.
- **протоиерей Андрей Ткачёв** — проповедник. Лексика: спасение, покаяние, благодать, литургия, Промысл Божий, святоотеческое предание, имена святых, библейские имена, названия икон.

## Среда исполнения

Ты работаешь в Claude Code на машине пользователя: есть локальная ФС, обычно GPU и интернет. Тяжёлый ASR гонишь **здесь, локально**. Не путать с песочницей claude.ai/Cowork (там сети/GPU/аудио-входа нет).

## Раскладка репозитория

```
audio/raw/        # исходники, как пришли
audio/wav/        # 01.wav … 15.wav после нормализации (16k mono pcm16)
refs/             # ref_malofeev.wav, ref_dugin.wav, ref_tkachev.wav (20–30с чистой речи)
out/asr/          # NN_raw.json (слова+таймкоды+спикер), NN_speaker.txt
out/clean/        # NN_clean.md (стадия A, источник истины)
out/final/        # NN_final.md (стадия B, литература) + NN_ratio.txt
glossary.md       # канонические написания (ведёшь и обновляешь по ходу)
scripts/          # run_asr.py и пр.
.env              # секреты (HF_TOKEN и т.п.) — НИКОГДА не коммить
```
Именование строго по плановому порядку: `01_<тема>.md … 15_<тема>.md`.

## Жёсткие правила (соблюдать всегда)

1. **Не выдумывать.** Не добавляй фактов, мыслей, примеров или «связок» с новым смыслом, которых не было в источнике. Сомнительное помечай `[?]`, не угадывай.
2. **Не сокращать по смыслу.** Удалять можно только слова-паразиты, заикания, ложные старты, артефакты ASR. Любое содержание сохраняется.
3. **Глоссарий — закон.** Термины и имена пишутся только по `glossary.md`. Нашёл новый/спорный термин — добавь кандидатом, спроси пользователя, зафиксируй канон.
4. **Контроль усадки обязателен** (см. стадию B). Это структурная защита, а не пожелание.
5. **Секреты.** Токены/ключи только из `.env`/переменных окружения. Не хардкодить, не коммитить, не печатать в лог. `.env` — в `.gitignore`.
6. **Спрашивай перед** необратимым: удаление данных, перезапись готовых `out/clean`/`out/final`, любые `git push`, отправка данных во внешние сервисы.
7. **Порядок бесед не менять.**

## Рабочий процесс

### Шаг 0 — окружение
Подними venv, поставь зависимости. Pyannote — gated: попроси пользователя положить `HF_TOKEN` в `.env` (сам токен не придумывай).
```bash
python -m venv .venv && . .venv/bin/activate
pip install whisperx faster-whisper pyannote.audio torch soundfile scipy numpy
# ffmpeg должен быть в системе
```

### Шаг 1 — подготовка аудио
Нормализуй каждый файл в `audio/wav/NN.wav`:
```bash
ffmpeg -i audio/raw/NN.* -ac 1 -ar 16000 -af loudnorm -c:a pcm_s16le audio/wav/NN.wav
```
Агрессивный деноиз не применяй — он роняет качество ASR. При необходимости — мягкий.

### Шаг 2 — ASR + диаризация с привязкой к именам (local-first)
Используй `scripts/run_asr.py`: WhisperX (large-v3) для распознавания и word-level выравнивания + pyannote 3.1 для диаризации с `min_speakers=max_speakers=3`. Кластеры переименовывай в **имена** по эталонным эмбеддингам из `refs/` (cosine), а не «SPEAKER_00».
- Биасинг: в `initial_prompt` передай имена/термины из профилей спикеров и глоссария.
- `condition_on_previous_text=False` (иначе на терминах Дугина модель зацикливается).
- (Улучшение, если стоит) основной русский ASR — **GigaAM-v2**, WhisperX как кросс-чек/выравнивание; на спорных сегментах — сверка двух гипотез.

Выход на беседу: `out/asr/NN_raw.json` и `out/asr/NN_speaker.txt` в формате `[мм:сс] Имя: реплика`.

**Калибровка:** сначала прогони ТОЛЬКО `01.wav`, покажи `01_speaker.txt`, дождись подтверждения качества (спикеры, термины). Потом — остальные 14.

### Шаг 3 — стадия A: точная редактура (verbatim) → `out/clean/NN_clean.md`
Для каждой беседы примени правила (это и есть твоя процедура редактирования):
> Приведи расшифровку к точному verbatim: убери слова-паразиты, заикания, ложные старты, артефакты ASR; исправь ошибочно распознанные термины и имена строго по глоссарию; восстанови пунктуацию и абзацы; проверь корректность меток спикеров по содержанию и стилю; сохрани таймкоды в начале реплик; ничего по смыслу не сокращай и ничего не дописывай; неуверенные места помечай `[?]`.

`NN_clean.md` — источник истины. С ним потом сверяется стадия B.

### Шаг 4 — стадия B: литературная редакция → `out/final/NN_final.md`
**Режь `NN_clean.md` на чанки** по репликам/смысловым блокам (~800–1500 токенов) и обрабатывай чанк за чанком. Это лечит усадку «10 страниц → 3» структурно: на маленьком куске модель не может «зарезюмировать всё».

Правила на чанк:
> Преобразуй фрагмент в связный, литературно выверенный текст. Сохрани весь смысл, все содержательные детали, авторский стиль каждого говорящего и порядок мыслей. Запрещено: добавлять то, чего не было; удалять содержание; менять термины/имена вопреки глоссарию.

**Контроль усадки (обязательно):** для каждого чанка считай `ratio = len(out_без_пробелов) / len(in_без_пробелов)`. Если `ratio < 0.85` — пометь `⚠ REVIEW`, переделай или вынеси на ручную проверку. Итог по файлу запиши в `out/final/NN_ratio.txt` (по чанкам + среднее). Общий объём `NN_final.md` ≈ объёму `NN_clean.md`.

Как именно выполнять стадию B:
- на первой беседе сделай правку сам в сессии (калибровка стиля);
- для масштаба можешь написать скрипт, вызывающий Claude API по чанкам с правилами выше как system-промптом (ключ — из `.env`, не коммить); подтверди подход у пользователя.

### Шаг 5 — сборка
- Шапка каждого файла: тема, дата, спикеры, исходный аудиофайл.
- Собери `out/final/00_оглавление.md` со ссылками на 15 файлов в плановом порядке.
- Структура сборника: ценности, которых **нет** в Указе № 809 (брошюра с ценностями Указа — отдельно, не трогаем).
- Финальную «литературную вычитку» целиком делает человек (Михаил), без сверки с аудио — это не твой шаг, просто подготовь к ней.

## Чек-лист приёмки (по каждой беседе)
- [ ] Спикеры названы именами, реплики не перепутаны.
- [ ] `NN_final.md` ≈ `NN_clean.md` по объёму; ни одного `⚠ REVIEW`.
- [ ] Все термины/имена — по `glossary.md`; меток `[?]` не осталось.
- [ ] Нет дописанного; спорное сверено с `NN_clean.md`.
- [ ] Сохранён индивидуальный голос каждого из троих.
- [ ] Файл назван и расположен по плановому порядку.
